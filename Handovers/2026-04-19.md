## Phase 0.5 changes

### Architecture changes
- `signals/domains.py` — shared LIFE_DOMAINS ontology, used by all detectors (eliminates inconsistent keyword lists)
- All detectors now emit `effect_size` and `data_quality` in signal metadata
- Pack generator passes data quality to LLM; LLM instructed to hedge marginal signals and refuse cross-signal synthesis when majority are marginal

### Detector fixes
| Detector | Change |
|----------|--------|
| `tone_shifts` | Added pre-flight: skips domain if <10% of messages score non-zero sentiment. Threshold raised to 0.15. Punctuation stripping added to `_sentiment_score` (was blocking most matches). |
| `absence` | Reframed finding text for Claude-only data: "not raised with AI assistant" not "absent from life". Marked as `data_quality: marginal` for single-source runs. |
| `frequency_salience` | Uses shared domains. Removed personal-context keywords (bunnings, etc). |

### New detectors
- `signals/validation_seeking.py` — detects "right?", "does that make sense?", "am I overthinking" patterns. Rate-based, per-topic breakdown. Genuinely unself-reportable.
- `signals/help_seeking.py` — classifies each message as practical / interpretive / emotional / validating. Flags emotional absence (<2%) and validating dominance (>15%). Strongest new signal.

## What Opus review flagged (addressed and deferred)

**Addressed in Phase 0.5:**
- Inconsistent domain ontologies → shared `domains.py`
- `tone_shifts` producing noise at delta < 0.10 → pre-flight quality gate + threshold 0.15
- `absence` claiming life-domain absence from AI-only data → reframed finding text
- Pack narrating noise as finding → quality-aware prompting
- Personal-context keywords (bunnings) → removed

**Deferred to Phase 1:**
- `abandoned_threads` needs complete rethink (design broken, not calibration)
- `contradiction` needs negation handling and more claim patterns
- Statistical significance testing for tone_shifts (bootstrap CIs)
- Attribution style signal (requires grammar-aware detection, not keyword matching)
- `abandoned_threads` replacement with first-person future-intent + resolution tracking
