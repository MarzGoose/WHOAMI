Three architectural approaches

The core question: what technology stack powers the engine and the desktop shell?

A — Tauri + Rust

Rust handles all file parsing, format detection, and signal extraction. Tauri provides the desktop shell. Tiny binary, no Chromium bloat, best performance.

For

  • Smallest app (~8MB vs Electron's ~150MB)
  • Strongest privacy/security story
  • Best long-term foundation
  • CLI comes naturally

Against

  • Rust is hard — slower to build
  • Longer time to first working version
Electron
Desktop Shell
+
Node.js
Core Engine
~150MB app · fast to build · massive ecosystem

B — Electron + Node.js

Node.js handles parsing (SQLite, CSV, JSON, XML). Electron wraps it in a desktop app. VS Code and Slack are built this way — proven, well-understood pattern.

For

  • Fastest path to working product
  • Huge parser ecosystem (npm)
  • Easy to hire for / open source
  • CLI straightforward

Against

  • Large install size (~150MB)
  • Higher memory use

C — Python core + Electron shell ★ recommended

Python does the heavy lifting: data parsing, format detection, pattern extraction (pandas, SQLite, spaCy for tone). Electron is a thin UI shell. The CLI is just the Python core run directly — no extra work.

For

  • Python is unmatched for data parsing
  • CLI is free — same codebase
  • Easier to prototype and iterate
  • AI/ML libraries available if needed later

Against

  • Two language contexts (Python + JS)
  • Slightly more complex IPC bridge

My recommendation is C — Python's data ecosystem is the best fit for this problem, and the CLI comes for free.
